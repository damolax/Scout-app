SCOUT V10.35.1 — RENDER SCALE GUARD WORKER

This worker is required after v10.35.1 is deployed. It replaces the old system where every open browser checked schedules every five seconds.

CREATE OR UPDATE THE RENDER SERVICE

1. Open Render.
2. Create a Background Worker from:
   https://github.com/damolax/Scout-app
3. Branch: main
4. Root directory: leave blank
5. Runtime: Node
6. Build command:
   npm config set registry https://registry.npmjs.org/ && npm ci --legacy-peer-deps --registry=https://registry.npmjs.org/
7. Start command:
   npm run worker:scale-guard

RENDER ENVIRONMENT VARIABLES

SCOUT_APP_URL=https://YOUR_REAL_SCOUT_DOMAIN
SCHEDULE_WORKER_SECRET=THE_EXACT_SAME_LONG_SECRET_USED_IN_VERCEL
SCOUT_WORKER_INTERVAL_MS=10000
SCOUT_WORKER_CAMPAIGNS_PER_TICK=6
SCOUT_WORKER_MAX_BACKOFF_MS=60000
NODE_VERSION=24

VERCEL MUST HAVE THE SAME SECRET

SCHEDULE_WORKER_SECRET=THE_EXACT_SAME_LONG_SECRET

VERIFY

The Render log should show:
Scout Scale Guard worker started.

When campaigns exist, it can show:
Processed 1 campaign pass(es).

If the log says SCHEDULE_WORKER_SECRET is required, add the missing variable and redeploy the worker.
If the log returns HTTP 401, the Render and Vercel secrets do not match.
If the log returns an RPC/schema-cache error, run RUN_THIS_SQL_FIRST_V10_35_1_SCALE_GUARD.sql and restart the worker.
